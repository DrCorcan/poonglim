1. 시스템 개요

풍림전자 A/S 및 제품 회수 업무는 Google Sheets를 원본 DB로 사용하고, ops.corcan.co.kr 웹 화면에서 조회·입력·수정합니다.

Google Sheets원본 데이터 저장소(시트명: 회수및AS입력장). 경영관리실이 최종 관리합니다.
ops.corcan.co.kr직원이 접속하는 웹 현황판. 입력, 조회, 다운로드를 수행합니다.
Apps Script웹 화면의 입력값을 Google Sheets에 저장하고 데이터를 불러옵니다.
이 매뉴얼의 설계 철학: 과거에는 택배로 제품이 들어와도 무엇인지, 몇 개인지 아무도 몰랐고 A/S 담당자 눈에 보이면 처리되고 안 보이면 누락되는 일이 있었습니다. 이 시스템의 목적은 들어온 건은 하나도 빠짐없이 화면에 보이게 하고, 처리가 끝날 때까지 계속 보이게 유지하는 것입니다. 반품·교환·A/S라는 구분보다 "아직 끝나지 않았다"는 사실 자체가 더 중요합니다.

2. 로그인·권한

웹 현황판은 Google 계정 자동 로그인이 아니라, USER_MASTER 시트의 이메일과 접속코드로 인증합니다.

USER_MASTER 컬럼설명
이메일직원 로그인 이메일
이름접수자·수정자·LOG에 표시될 이름
사용여부Y이면 사용 가능, N이면 접속 차단
접속코드입력값과 시트 값이 정확히 일치하면 로그인 가능
직원 퇴사 또는 권한 중지는 USER_MASTER의 사용여부를 N으로 바꾸면 됩니다.

3. Google Sheets 원본 데이터 구조

원본 시트명은 회수및AS입력장입니다. 현재 기준 컬럼은 A~AC, 총 29개를 사용합니다.

범위주요 컬럼설명
A~Q접수일, 구분, 채널, 고객명, 연락처, 상품명, 수량, 주문번호, 회수송장번호, 택배접수일, 택배입고일, CS내역, 불량여부, 처리결과, ERP처리, 완료여부, 비고업무 기본 정보
R~U수리비용, 입금확인, 출고송장, 수리완료일정산·출고·완료 관리
V~Y접수자, 접수자이메일, 최종수정자, 최종수정시간자동 기록 영역
Z주소사방넷 업로드 양식의 주소로 사용
AA택배비택배비만 청구하는 케이스 및 사방넷 택배비로 사용
AB택배사CJ대한통운 / 롯데택배 / 한진택배 / 우체국택배 / 로젠택배 / 기타 중 선택
AC입금확인일입금확인 처리한 날짜가 자동 기록됨 (7장 참고)
중간에 컬럼을 삽입하면 프로그램 컬럼 번호가 전부 밀려서 오작동합니다. 새 항목은 반드시 맨 오른쪽(다음 빈 컬럼)에 추가하고, Apps Script의 COL 번호와 TOTAL_COLS도 함께 수정해야 합니다.

4. 현황판 보는 법

현황판(📊 현황 탭)은 위에서 아래로 오늘 현황 → 전체 작업 대기 현황 → 누적 통계 / 오늘 할 일 순서로 구성되어 있습니다. 반품·교환·A/S 구분 없이 처리 단계 기준으로 통합되어 있는 것이 특징입니다.

① 오늘의 접수·처리 현황 (맨 위, 5칸)

카드기준
오늘 접수사방넷 접수일이 오늘인 건
오늘 입고택배입고일이 오늘인 건
오늘 출고출고일/수리완료일이 오늘인 건
오늘 처리완료출고일이 오늘인 완료 건, 또는 출고 없이 그 자리에서 끝나는 처리결과(양품화완료·불량보관·본사내방수리완료·폐기·환불처리·고객협의완료)로 오늘 완료된 건. A/S는 수리 후 출고 없이 그 자리에서 끝나는 경우가 많아 출고일만으로는 놓칠 수 있음
오늘 개인고객 입금확인입금확인일이 오늘이면서 청구유형이 '개인고객 청구'인 건 (거래처 월정산은 제외)

이 카드들은 구분(반품/교환/A/S) 전체를 포함하고, 접수일이 오래된 건이라도 오늘 입고/출고/입금확인된 것이면 빠짐없이 잡힙니다.

② 처리 대기 목록 (구분 통합 작업판, 5칸)

카드의미
미입고 (회수/택배 대기)완료여부 = 진행중. 아직 택배가 도착하지 않은 건
입고완료 판정/처리 필요완료여부 = 입고완료/처리대기. 양품·불량을 정하고 처리방법을 선택해야 하는 건
출고대기 (처리결과 입력됨)완료여부 = 출고대기. 처리방법은 정했고 출고송장만 입력하면 끝나는 건
청구/입금 확인 필요수리비용 또는 택배비가 발생했고, 아직 입금확인되지 않은 건 (7장 참고)
처리완료완료여부 = 완료. 반품/교환은 ERP처리까지 Y인 건만, A/S는 완료 상태면 바로 포함
이 카드들은 연도·기간 제한이 없습니다. 25년 건이든 26년 건이든, 처리가 끝나지 않은 건은 절대 사라지지 않고 계속 잡힙니다. "최근 90일"이라는 제한은 ⑧ 전체 목록 탭에만 적용됩니다.

③ 26년 누적 — 구분별 현황 (참고용)

반품·교환·A/S가 각각 몇 건인지 보여주는 작은 참고용 통계입니다. 26년 접수 건만 집계하며, 작업 대기 여부와는 무관합니다.

④ 오늘 할 일

판정/처리 필요 건수와 출고대기 건수를 합친 숫자 하나만 보여줍니다. 클릭하면 전체 목록 탭으로 이동해 실제 처리 대상 목록이 펼쳐집니다. A/S 직원은 화면 최상단 우측(오늘 개인고객 입금확인)과 최하단 우측(오늘 할 일)만 확인하면 됩니다.

6. 처리 입력

7. 청구·입금 관리

유상 A/S 또는 택배비 청구가 필요한 경우 수리비용 또는 택배비를 입력합니다. 받는 대상이 거래처(월 단위로 모아서 정산)인지 개인 고객(건건이 즉시 결제)인지를 처음 청구 단계에서 구분합니다.

입금확인(S열) 값의미현황판 청구/입금 포함 여부
거래처 청구거래처에 월 1~2회 모아서 청구할 대상. 입금 전까지 이 값이 유지됨입금확인일이 없으면 포함
개인고객 청구개인 고객에게 건건이 안내해서 받을 대상. 입금 전까지 이 값이 유지됨입금확인일이 없으면 포함
입금확인화면에서 선택하면 청구유형(거래처/개인고객)은 그대로 두고 AC열 입금확인일에 오늘 날짜만 기록됨제외
무상청구할 비용 없음제외
관리제외과거/이관 데이터 등 청구 관리 대상 제외제외
청구/입금 대상 조건: (수리비용 + 택배비) > 0 그리고 입금확인일(AC열)이 비어 있음 그리고 청구유형이 무상 / 관리제외가 아님
핵심 설계: 입금이 끝나도 "이게 거래처 건이었는지 개인 건이었는지"는 S열에 그대로 남습니다. 실제 입금 여부는 오직 AC열(입금확인일)의 유무로만 판단합니다. 그래서 입금확인 처리 후에도 과거 기록을 보면 어떤 유형이었는지 알 수 있습니다.

일괄 입금확인 (거래처 월정산용)

현황판의 "청구/입금 확인 필요" 카드를 클릭하면 일반 목록이 아니라 체크박스가 있는 일괄확인 화면이 열립니다.

1
청구유형 필터 선택
상단의 전체 / 거래처 청구 / 개인고객 청구 탭으로 좁힙니다.
2
체크박스로 건 선택
거래처에서 입금받을 건들을 체크합니다. 화면 하단에 선택 건수와 합계 금액(수리비용+택배비)이 실시간으로 표시됩니다.
3
합계 대조
거래처가 실제로 입금한 금액과 화면 하단 합계가 일치하는지 확인합니다.
4
일괄 처리
금액이 맞으면 "✅ 선택 항목 입금확인 처리" 버튼을 눌러 체크된 모든 건을 한 번에 입금확인 처리합니다.

개인 고객 건은 보통 들어올 때마다 한 건씩 체크해서 바로 처리하면 됩니다.

9. 엑셀 다운로드

전체 목록에서 원하는 필터를 선택한 뒤 다운로드하면 현재 조건에 맞는 데이터가 내려받아집니다.

현재 목록 엑셀화면에 필터된 목록을 원본 컬럼 전체 기준(택배사·입금확인일 포함)으로 다운로드합니다. LOG 이력은 제외됩니다.

택배비만 청구하는 경우: 수리비용은 빈칸 또는 0, 택배비만 입력하면 결제가격은 택배비로 계산됩니다.

12. 연도별 데이터 정리 (매년 반복 작업)

매출 회전을 위해 지난 연도 데이터는 별도 보관 탭으로 옮기고 원본에서는 행을 삭제합니다. 매년 초 동일한 절차로 반복합니다.

1
LOG 시트를 헤더만 남기고 비웁니다.
LOG는 주문번호 기준으로 기록되므로 행 삭제 자체로는 깨지지 않지만, 연도 전환 시점에 깨끗이 정리하는 것을 권장합니다.
2
원본 시트(회수및AS입력장)를 시트 복사로 통째로 복제
새 탭 이름을 보관_2025 형식으로 짓습니다.
3
복사본에서는 지난 연도 데이터만 남기고 나머지 삭제
4
원본에서는 지난 연도 데이터만 삭제하고 올해 데이터만 남깁니다.
이 단계가 실제로 행이 사라지는 단계이므로 가장 마지막에 진행합니다.
LOG가 주문번호 기준(행 번호 기준 아님)으로 기록되기 때문에, 이 작업으로 행 위치가 바뀌어도 이후 쌓이는 로그 조회에는 영향이 없습니다.
보관 탭으로 옮긴 데이터는 조회만 가능하며, 현황판(ops.corcan.co.kr)에서는 보이지 않습니다. 옛 데이터를 확인하려면 Google Sheets를 직접 열어야 합니다.

14. 문제 해결

문제확인 방법
로그인이 안 됨USER_MASTER의 이메일, 사용여부 Y, 접속코드 확인. 직원 브라우저의 사이트 데이터 삭제 후 재로그인.
데이터가 최신이 아님전체 목록 탭/필터/다운로드 시 자동 최신화됩니다. 그래도 이상하면 상단 새로고침 또는 Ctrl+F5.
청구/입금 숫자가 이상함수리비용 또는 택배비가 있는지, 입금확인일(AC열)이 비어있는지, 청구유형이 무상/관리제외로 되어 있지 않은지 확인.
오늘 들어온 건이 카드에 안 보임구분(반품/교환/A/S)별로 카드가 나뉘어 있던 과거 화면과 달리, 지금은 "처리 대기 목록"이 구분 통합이라 한 카드에 다 같이 잡힙니다. 카드를 클릭해 목록에서 배지(반품/교환/A/S)로 구분을 확인하세요.
로그보기에 과거 이력이 안 보임LOG는 주문번호 기준입니다. 주문번호가 없는 A/S 상담접수 건은 '주문번호없음:고객명' 형태로 기록되므로, 고객명이 바뀌면 과거 로그와 매칭되지 않을 수 있습니다.
사방넷 양식 주소가 비어 있음Z열 주소가 입력되어 있는지 확인.
완료 건 수정 필요전체 목록에서 완료 필터 선택 후 수정 ▶ 버튼으로 수정.
Google Sheets 원본 데이터를 직접 수정할 때는 컬럼 위치를 임의로 바꾸지 마세요. 프로그램 저장 위치와 다운로드 매핑이 달라질 수 있습니다. 새 항목 추가는 항상 맨 오른쪽 컬럼에서만 진행합니다.
