1. 시스템 개요

풍림전자 A/S 및 제품 회수 업무는 Google Sheets를 원본 DB로 사용하고, ops.corcan.co.kr 웹 화면에서 조회·입력·수정합니다.

Google Sheets원본 데이터 저장소. 경영관리실이 최종 관리합니다.
ops.corcan.co.kr직원이 접속하는 웹 현황판. 입력, 조회, 다운로드를 수행합니다.
Apps Script웹 화면의 입력값을 Google Sheets에 저장하고 데이터를 불러옵니다.

2. 로그인·권한

웹 현황판은 Google 로그인 이메일 자동 확인이 아니라, USER_MASTER 시트의 이메일과 접속코드로 인증합니다.

USER_MASTER 컬럼설명
이메일직원 로그인 이메일
이름접수자·수정자·LOG에 표시될 이름
사용여부Y이면 사용 가능, N이면 접속 차단
접속코드4자리일 필요 없음. 입력값과 시트 값이 정확히 일치하면 로그인 가능
직원 퇴사 또는 권한 중지는 USER_MASTER의 사용여부를 N으로 바꾸면 됩니다.

3. Google Sheets 원본 데이터 구조

원본 시트명은 회수및AS입력장입니다. 현재 기준 컬럼은 A~AA까지 사용합니다.

범위주요 컬럼설명
A~Q접수일, 구분, 채널, 고객명, 연락처, 상품명, 주문번호, 회수송장, CS내역, 처리결과, ERP처리, 완료여부, 비고업무 기본 정보
R~U수리비용, 입금확인, 출고송장, 수리완료일정산·출고·완료 관리
V~Y접수자, 접수자이메일, 최종수정자, 최종수정시간자동 기록 영역
Z주소사방넷 업로드 양식의 주소로 사용
AA택배비택배비만 청구하는 케이스 및 사방넷 택배비로 사용
중간에 컬럼을 삽입하면 프로그램 컬럼 번호가 밀릴 수 있습니다. 새 항목은 가능하면 맨 오른쪽에 추가합니다.

4. 현황판 보는 법

현황판은 반품/교환, A/S, 오늘 처리할 일을 카드 숫자로 보여줍니다. 숫자를 누르면 해당 조건의 전체 목록으로 이동합니다.

카드의미
회수 미입고접수되었지만 제품 입고가 아직 확인되지 않은 건
입고완료/처리대기제품은 입고됐지만 처리결과 입력이 필요한 건
A/S 청구/입금수리비용 또는 택배비가 있고, 아직 입금확인/무상/관리제외가 아닌 건
완료처리결과 또는 출고송장 기준으로 완료된 건

전체 목록, 필터, 다운로드 시 최신 데이터를 자동 재조회하도록 되어 있습니다.

6. 처리 입력

처리 입력 화면은 제품 확인 후 실제 조치 내용을 기록하는 곳입니다. 완료 건도 전체 목록에서 수정 ▶ 버튼으로 다시 수정할 수 있습니다.

9. 엑셀 다운로드

전체 목록에서 원하는 필터를 선택한 뒤 다운로드하면 현재 조건에 맞는 데이터가 내려받아집니다.

현재 목록 엑셀화면에 필터된 목록을 원본 컬럼 전체 기준으로 다운로드합니다. LOG 이력은 제외됩니다.

13. 문제 해결

문제확인 방법
로그인이 안 됨USER_MASTER의 이메일, 사용여부 Y, 접속코드 확인. 직원 브라우저의 사이트 데이터 삭제 후 재로그인.
데이터가 최신이 아님전체 목록 탭/필터/다운로드 시 자동 최신화됩니다. 그래도 이상하면 상단 새로고침 또는 Ctrl+F5.
청구/입금 숫자가 이상함수리비용 또는 택배비가 있는지, 입금확인이 입금확인/무상/관리제외로 되어 있지 않은지 확인.
사방넷 양식 주소가 비어 있음Z열 주소가 입력되어 있는지 확인.
완료 건 수정 필요전체 목록에서 완료 필터 선택 후 수정 ▶ 버튼으로 수정.
Google Sheets 원본 데이터를 직접 수정할 때는 컬럼 위치를 임의로 바꾸지 마세요. 프로그램 저장 위치와 다운로드 매핑이 달라질 수 있습니다.
