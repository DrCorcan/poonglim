📌 핵심 원칙 사방넷의 출고대기 상태인 건만 캐리어에 실제 발주됩니다.
신규주문/주문확인 상태는 해피콜대기로 보관되며, 해피콜 완료 후 출고대기로 바뀌면 자동으로 발주 대상이 됩니다.

참고

완료여부 상태값 체계

구글시트 캐리어_발주원장의 완료여부 컬럼은 각 주문의 현재 처리 단계를 나타냅니다.

상태값 의미 발생 조건 재고 영향 다음 단계
해피콜대기 신규주문/주문확인 — 해피콜 전 매입처발주.py 자동 없음 해피콜 완료 → 출고대기 → 재실행 시 신규
신규 발주서 생성 대상 매입처발주.py 자동 없음 발주서생성.py 실행 → 발주중
기존재고 선매입재고 사용 (캐리어 발주 불필요) 매입처발주.py 자동 선매입재고 -1 오더번호 자동 입력됨
발주중 발주서 발송 완료, 오더번호 대기 발주서생성.py 자동 없음 캐리어 회신 후 업데이트.py → 발주완료
발주완료 오더번호 입력 완료 업데이트.py 자동 없음 처리 완료
주문취소 캐리어 발주 전 취소 현황판 취소처리 탭 없음 처리 완료
취소재고 캐리어 발주 후 취소 현황판 취소처리 탭 선매입재고 +1 복구 처리 완료
⚠️ 취소 처리 핵심 기준 오더번호가 없으면 → 주문취소 (재고 무영향)
오더번호가 있으면 → 취소재고 (선매입재고 복구)
1단계

발주총괄 → 구글시트 업로드

실행 파일

  • 사방넷에서 발주총괄 다운로드 (파일명에 '발주총괄' 포함)
  • 파일을 1단계_발주원장구글시트업로드\원본\ 폴더에 넣기
  • 매입처발주_실행.bat 더블클릭

자동 처리 내용

🔔

신규주문 / 주문확인 → 해피콜대기

사방넷에서 아직 출고대기가 아닌 건. 해피콜 완료 전까지 보관됩니다.

🆕

출고대기 + 선매입재고 없음 → 신규

캐리어에 새로 발주해야 하는 건. 2단계에서 발주서가 생성됩니다.

📦

출고대기 + 선매입재고 있음 → 기존재고

이미 설치점에 보관 중인 재고로 처리. 캐리어 발주 불필요. 오더번호 자동 입력.

✅

해피콜대기 건이 출고대기로 바뀐 경우 → 신규 자동 전환

매입처발주.py 재실행 시 해피콜 완료 후 출고대기가 된 건을 자동으로 신규로 변경합니다.

💡 발주총괄은 매일 최신 파일로 실행하세요. 재실행해도 중복 등록되지 않습니다. 상태가 바뀐 건만 업데이트됩니다.
2단계

발주서 생성 → 캐리어 이메일 발송

실행 파일

  • 2단계_발주서생성\발주서생성_실행.bat 더블클릭
  • 생성된 캐리어_발주서_YYYYMMDD.xlsx를 캐리어에 이메일 발송 (수동)

발주서 자동 설정 내용

항목규칙
J열 설치비포함설치점 배송 → Z01(설치비포함) / 직송·택배 → Z02(설치비미포함)
납품요청일배송: 발주일 +1일 / 설치: 월화→수, 수목→금, 금→월
행 정렬모델코드 기준 오름차순 정렬
AC열 비고1배송건: 물류메시지 / 설치건: 설치점명
AD열 비고2설치건: 물류메시지 / 배송건: 공란
✅ 발주서 생성 후 자동 처리 구글시트 캐리어_발주원장의 완료여부가 신규 → 발주중으로 자동 변경됩니다. 현황판에서 신규 카운트가 0이 되면 정상입니다.
⚠️ 발주서 재생성이 필요한 경우: 구글시트에서 해당 건의 완료여부를 발주중 → 신규로 수동 변경 후 다시 실행하세요.
3단계

오더번호 업데이트

캐리어에서 오더번호가 포함된 회신 파일이 오면 자동으로 구글시트에 입력합니다.

실행 방법

  • 캐리어 회신 파일(풍림발주서_*캐리어*.xlsx)을 3단계_발주원장업데이트\원본\에 넣기
  • 캐리어오더번호업데이트_실행.bat 더블클릭

매칭 기준 (3단계)

순위기준설명
1순위핸드폰번호 + 모델코드가장 일반적인 경우
2순위일반전화번호 + 모델코드캐리어가 다른 전화번호로 입력한 경우
3순위주소 앞 20자 + 모델코드캐리어가 전화번호를 잘못 입력한 경우
💡 매칭 성공 시 완료여부가 발주중 → 발주완료로 자동 변경됩니다.
매칭 실패 건은 ⚠️ 경고로 출력됩니다 → 현황판 오더번호 미입력 탭에서 수동 입력하세요.
ℹ️ 같은 파일을 다시 실행해도 괜찮습니다 이미 발주완료 처리된 건은 "ℹ️ 이미 처리됨"으로 안내되고 중복 처리되지 않습니다.
현황판

현황판 보는 법

현황판은 4단 구조로 구성되어 있습니다.

1단 — 캐리어 에어컨 발주 현황

전체 누적 발주 현황입니다. 카드를 클릭하면 해당 상태의 발주 목록으로 이동합니다.
🔔

해피콜대기

신규주문/주문확인 상태인 건. 해피콜 전 대기 중. 클릭하면 전용 처리 화면으로 이동.

🆕

신규

오늘 발주서를 생성해야 하는 건. 2단계 실행이 필요합니다.

🔴

오더번호 미입력

발주서를 보냈는데 캐리어 회신이 없는 건. 클릭하면 수동 입력 화면으로 이동.

✅

발주완료

오더번호까지 정상 처리된 건. 정상 상태입니다.

🟡

기존재고

선매입재고로 처리된 건. 캐리어 발주 없이 기존 재고 사용.

📦

가용재고

현재 설치점에 대기 중인 선매입재고 수량. 클릭하면 모델별/설치점별 세부 내역 조회.

2단 — 오늘의 주문 접수 현황

오늘 접수된 주문만 집계합니다. 매입처발주.py 실행 후 오늘 처리 결과를 확인할 때 사용합니다.

3단 — 이번달 발주 현황

이번달 발주완료 기준 건수와 금액(VAT포함)을 보여줍니다.

4단 — 취소 및 CS 내역

🚫

주문취소

캐리어 발주 전 취소된 건. 재고에 영향 없음.

❌

취소재고

캐리어 발주 후 취소된 건. 선매입재고로 복구됨.

🚨

CS 진행

CS여부=Y로 표시된 건. 클릭하면 CS 입력 화면으로 이동.

방법 1 — 현황판에서 직접 취소

FAQ

자주 묻는 질문

Q. 오더번호 미입력이 0인데 실제로 발주중 건이 있어요

  • Apps Script를 최신 버전으로 재배포했는지 확인하세요
  • 현황판 새로고침 버튼(🔄)을 눌러보세요

Q. 발주서를 잘못 생성했어요 (재생성 필요)

  • 구글시트 캐리어_발주원장에서 해당 건 완료여부를 발주중 → 신규로 수동 변경
  • 발주서생성_실행.bat 다시 실행

Q. 캐리어 회신파일에서 매칭 실패가 떴어요

  • 현황판 🔴 오더번호 미입력 카드 클릭 → 수동 입력
  • 또는 구글시트에서 직접 오더번호 입력
  • 캐리어가 전화번호를 잘못 입력한 경우 주소 매칭(3순위)으로 처리됨

Q. 같은 발주총괄 파일을 실수로 두 번 실행했어요

  • 중복 체크가 되어 있어 걱정하지 않아도 됩니다
  • 이미 등록된 사방넷주문번호는 건너뜁니다

Q. 발주현황 탭에서 건수가 구글시트와 다르게 보여요

  • 날짜 필터를 확인하세요 (기본값: 전체 기간)
  • 발주현황 탭 진입 시 자동으로 최신 데이터를 불러옵니다
