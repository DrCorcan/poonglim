# 풍림전자 매출관리포털 AI 인수인계 문서

이 문서는 새 AI 에이전트 또는 새 ChatGPT 대화창에서 풍림전자 매출관리포털 프로젝트를 이어갈 때 가장 먼저 읽어야 하는 요약 지침서다.

---

## 1. 프로젝트 목적

풍림전자 사방넷 주문 처리 업무를 파일 중심에서 원장 중심으로 전환한다.

기존 흐름:

```text
사방넷 엑셀 다운로드
→ 파이썬 실행
→ 오성 업로드 파일 생성
→ 오성 송장 다운로드
→ 파이썬 실행
→ 사방넷 송장 업로드 파일 생성
→ 거래처별 송장 회신 파일 생성
→ ERP 업로드 파일 생성
→ ERP 업로드
→ 사방넷/ERP 마감대조
```

개편 흐름:

```text
사방넷 주문자료를 매출관리원장에 1회 적재
→ 원장에서 오성 업로드 파일 생성
→ 원장에서 송장 수신/매칭
→ 원장에서 사방넷 송장업로드 파일 생성
→ 원장에서 거래처별 송장파일 생성
→ 원장에서 ERP 업로드 파일 생성
→ 원장에서 마감대조 실행
```

---

## 2. 전체 역할 구분

```text
사방넷 = 주문 수집 / 판매채널 / 송장 업로드
풍림전자 매출관리포털 = 업무 통제센터 / 원장 / 상태관리 / 파일 생성 / AI 에이전트 작업공간
ERP = 출고 / 반품 / 교환 / 재고 / 회계 / 결산 확정 시스템
AI 에이전트 = 포털 데이터를 보고 검수·추천·파일생성·예외확인
```

## 3. 절대 바꾸면 안 되는 기준

### 기준 1. 주문상세원장 1행 기준

```text
02_주문상세원장 1행 = 사방넷주문번호 1개
```

절대 `주문번호`를 원장 1행 기준으로 삼지 말 것. 주문번호는 묶음 주문 그룹값이다.

### 기준 2. 오성 송장 매칭 기준

```text
오성 송장자료.주문번호 = 주문상세원장.사방넷주문번호
```

### 기준 3. 거래처 송장 회신 기준

```text
거래처 송장 회신 기준 = 점포명
매입처 기준 아님
```

점포명 → 거래처_히스토리 → 거래처명/이메일 매핑 구조를 사용한다.

### 기준 4. 비오성 직행

```text
물류처 ≠ 오성물류
→ 창고업로드상태 = 제외
→ 1~3번 오성 단계를 건너뜀
```

### 기준 5. 오성 발주 후 취소 제동

```text
물류처 = 오성물류
AND 창고업로드상태 IN (파일생성완료, 업로드완료)
AND 현재주문상태가 취소/반품/교환 계열
→ 오류상태 = 확인필요
→ 오류유형 = 출고전취소
→ 제동여부 = Y
```

이 상태에서는 사방넷 송장업로드, 거래처 회신, ERP 업로드, 마감 정상 처리를 자동 진행하면 안 된다.

### 기준 6. ERP는 임시 설계

현재 ERP 업로드는 기존 주문등록 로직을 임시 기준으로 두고 있다. 향후 ERP 화면을 보고 출고/반품/교환 중심으로 재설계할 수 있다.

## 4. 구글시트 탭 구조

```text
01_사방넷원본수집
02_주문상세원장
03_창고업로드이력
04_송장수신이력
05_사방넷송장업로드이력
06_거래처회신이력
07_ERP업로드이력
08_마감대조원장
90_실행로그
91_오류로그
99_상태값
```

핵심:

```text
02_주문상세원장 = 현재 상태 판단 기준
03~08 = 이력/대조 결과 누적
90~91 = 시스템 로그 및 오류 로그
99_상태값 = 드롭다운/코드표 기준
```

## 5. 10개 버튼 흐름

```text
1. 오성 업로드 파일 생성
2. 오성 업로드 완료 처리
3. 오성 송장자료 업로드
4. 사방넷 송장업로드 파일 생성
5. 사방넷 송장 업로드 완료 처리
6. 거래처별 송장파일 생성
7. 거래처 메일 발송 완료 처리
8. ERP 업로드 파일 생성
9. ERP 업로드 완료 처리
10. 마감대조 실행
```

외부 시스템에 실제 반영되는 단계는 사람이 확인한다.

```text
파일생성완료 ≠ 업로드완료
```

오성 전산 업로드완료, 사방넷 송장 업로드완료, ERP 업로드완료는 자동으로 넘기지 않는다. 사람이 실제 반영을 확인하고 완료 처리 버튼을 눌러야 한다.

## 6. 오성 업로드 핵심 로직

대상:

```text
물류처에 오성 포함
현재주문상태 IN (신규주문, 주문확인)
창고업로드상태 = 대기
오류상태 = 정상
```

오성 파일 헤더:

```text
업체코드
업체명
주문번호1
주문번호2
주문번호3
수화인
주소
연락처
휴대전화
배송메시지
상품바코드
상품명
상품코드
상품코드2
컬러
사이즈
출고수량
```

중요 로직:

```text
상품코드테이블로 바코드 매핑
NS-3000 + 치마형 키트 합포장 처리
NS-3000 수량 2~3개는 1개씩 행 분리
주소 누락 시 파일 생성 중단
배송메시지는 빈칸
수화인 이름 정제
```

## 8. 현재까지 작성된 초기 세팅 코드

초기 구글시트 탭/헤더/서식/상태값 시트를 생성하는 Apps Script를 작성했다. 다음 개발 단계는 다음과 같다.

```text
1. 상태값 드롭다운 자동 적용
2. 사방넷 원본 업로드 → 02_주문상세원장 자동 반영 로직 작성
3. 오성 업로드 파일 생성 버튼 로직 작성
```

## 9. 새 AI 에이전트가 작업할 때 주의할 점

1. 먼저 `sales-portal-design.md` 전체를 읽는다.
2. 그 다음 `sales-portal-changelog.md`에서 최신 변경사항을 확인한다.
3. 사용자가 새 예외를 말하면 설계서와 변경이력 모두 업데이트한다.
4. 코드 작성 전 반드시 기준 키가 `사방넷주문번호`인지 확인한다.
5. 거래처 회신을 만들 때 매입처 기준으로 잘못 설계하지 않는다.
6. ERP는 아직 최종 확정이 아니므로 임의로 완성 설계처럼 단정하지 않는다.
7. 오성 발주 후 취소/반품 제동은 반드시 후속 자동처리보다 우선한다.
