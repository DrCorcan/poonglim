# 사방넷 발주서 변환 데이터 아키텍처

문서 버전: v1.0  
작성일: 2026-06-19

---

## 1. 데이터 아키텍처 목적

이 문서는 `sabang.py`가 어떤 입력 데이터를 읽고, 어떤 마스터를 참조하며, 어떤 출력 데이터를 생성하는지 정의한다.

핵심 목적은 다음이다.

```text
거래처별 발주서 양식이 달라도
거래처_매핑테이블과 거래처_단가테이블을 기준으로
표준 사방넷 입력본을 안정적으로 생성한다.
```

---

## 2. 데이터 흐름

```text
[거래처 발주서 원본]
    - xlsx / xls
    - 거래처별 양식 상이
    - 원본 폴더에 저장

        ↓ 파일명 키워드 식별

[거래처_매핑테이블.xlsx]
    - 거래처 식별
    - 원본 컬럼 위치 정의
    - 데이터 시작행 정의
    - 주문번호/옵션/필터 정의

        ↓ 원본 행 파싱

[거래처_단가테이블.xlsx]
    - 거래처별 상품코드 키워드
    - 공급단가
    - 택배비
    - 제주추가택배비
    - 사방넷모델명

        ↓ 변환

[검증 체크리스트]
    - 콘솔 출력
    - 거래처별 수취인/수량/모델명/단가/택배비/주문번호 확인

        ↓ 저장

[사방넷 입력본]
    - 완료/YYYYMMDD_거래처명_사방넷입력본.xlsx
```

---

## 3. 주요 엔티티

| 엔티티 | 설명 | 주요 키 |
|---|---|---|
| 거래처 발주서 원본 | 거래처가 보낸 주문 엑셀 | 파일명, 원본행번호 |
| 거래처 매핑 | 발주서 양식 해석 기준 | 거래처명, 파일명키워드 |
| 거래처 단가 | 상품 매칭/금액 기준 | 거래처명, 상품코드 |
| 변환 주문행 | 사방넷 입력본 1행 | 거래처명, 주문번호, 수취인, 모델명 |
| 사방넷 입력본 | 업로드 대상 파일 | 날짜, 거래처명 |
| 검증 로그 | 사람이 확인할 실행 결과 | 거래처명, 순번 |

---

## 4. 주요 키 설계

### 4-1. 거래처 식별 키

```text
파일명에 거래처_매핑테이블.파일명키워드가 포함되면 해당 거래처로 식별한다.
```

예:

```text
이데컴퍼니_풍림전자_발주서_양식_V20260618.xlsx
→ 파일명키워드: 이데컴퍼니
→ 거래처명: 이데컴퍼니
```

### 4-2. 컬럼 매핑 키

거래처별로 원본 발주서 컬럼 위치가 다르므로 `거래처_매핑테이블.xlsx`에서 각 항목을 열 문자로 지정한다.

예:

```text
B_수취인 = B
D_주소 = C
G_상품명 = F
H_모델명 = G
I_수량 = H
K_배송메시지 = J
P_주문번호 = I
```

### 4-3. 상품 매칭 키

상품 매칭은 `거래처_단가테이블.xlsx`의 `상품코드`를 기준으로 한다.

```text
검색텍스트 = 원본 모델명 + 원본 상품명 + 원본 옵션
```

검색텍스트 안에 단가테이블의 `상품코드`가 포함되면 해당 상품으로 매칭한다.

## 5. 입력 데이터

### 5-1. 원본 발주서

원본 폴더에 들어오는 거래처 발주서다.

```text
원본/*.xlsx
원본/*.xls
```

주의:

```text
.xls 파일 처리를 위해 PC에 xlrd 설치가 필요하다.
```

### 5-2. 거래처_매핑테이블.xlsx

역할:

```text
파일명으로 거래처 식별
원본 발주서 컬럼 위치 지정
데이터 시작행 지정
합계 행 제외 필터 지정
발주서 주문번호 사용 여부 지정
```

### 5-3. 거래처_단가테이블.xlsx

역할:

```text
거래처별 상품코드 키워드 관리
공급단가 관리
택배비 관리
제주추가택배비 관리
사방넷모델명 관리
```

---

## 6. 처리 데이터

### 6-1. 검색텍스트

```text
검색텍스트 = 모델명 + 상품명 + 옵션
```

예:

```text
원본 상품명: 펜소닉 듀라론 100% 냉감패드 PCN-2000 (더블)
원본 모델명: 빈칸
옵션: 빈칸
검색텍스트: 펜소닉 듀라론 100% 냉감패드 PCN-2000 (더블)
상품코드 PCN-2000 매칭
```

### 6-2. 사방넷모델명

단가테이블의 `사방넷모델명`이 있으면 최종 모델명으로 사용한다.

없으면 원본 모델명을 사용하지만, 이는 미매칭 가능성이 있으므로 검증 대상이다.

### 6-3. 공급가합

```text
공급가합 = 공급단가 × 수량
```

원본 발주서 공급가합을 사용하지 않는다.

## 8. 위험 지점

### 8-1. 기본단가로 잘못 빠지는 문제

현재 구조상 상품코드 매칭에 실패해도 거래처의 기본 공급단가/택배비로 출력될 수 있다.

따라서 아래 현상은 미매칭 의심이다.

```text
모델명이 사방넷모델명으로 바뀌지 않음
모델명이 원본 상품명 일부로 출력됨
새 상품인데 기존 거래처 첫 번째 단가가 출력됨
공급단가가 이상하게 낮거나 반복됨
```

### 8-2. 합계 행 포함 문제

일부 발주서는 마지막에 합계 행이 있다.
필터가 없으면 합계 행이 주문으로 처리될 수 있다.

예:

```text
주니조아 합계 행 포함
→ 업체필터컬럼 B / 업체필터값 선불로 해결
```

### 8-3. 시트 선택 경고

시트명이 당일 날짜와 맞지 않으면 첫 번째 시트를 사용한다.
실제 발주 데이터가 첫 번째 시트라면 문제가 아니다.

## 9. 매출관리포털과의 관계

이 변환기는 매출관리포털의 전단계 업무다.

```text
거래처 발주서 변환기
= 외부 발주서를 사방넷에 넣기 위한 입력본 생성

매출관리포털
= 사방넷에 들어간 이후 주문 원장, 출고, 송장, ERP, 마감 관리
```

두 흐름은 연결될 수 있지만 문서와 데이터 아키텍처는 분리한다.
