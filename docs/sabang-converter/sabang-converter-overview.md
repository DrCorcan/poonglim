# 사방넷 발주서 변환 개요

문서 버전: v1.0  
작성일: 2026-06-19  
대상 업무: 거래처별 발주서 → 사방넷 입력본 자동 변환

---

## 1. 목적

이 문서는 풍림전자 거래처별 발주서를 `sabang.py`로 변환하여 사방넷 업로드용 입력본을 생성하는 업무의 기준을 정의한다.

이 업무의 핵심은 단순히 파이썬 프로그램이 실행되는지 확인하는 것이 아니다.

핵심은 다음이다.

```text
현재 들어온 발주서가
거래처_매핑테이블.xlsx와 거래처_단가테이블.xlsx에 정확히 연결되어
사방넷에 업로드 가능한 입력본으로 정상 출력되는지 검증한다.
```

---

## 2. 전체 흐름

```text
거래처 발주서 원본
    ↓
원본 폴더
    ↓
sabang.py 실행
    ↓
거래처_매핑테이블.xlsx 기준으로 거래처/컬럼 식별
    ↓
거래처_단가테이블.xlsx 기준으로 상품코드/단가/택배비/사방넷모델명 매칭
    ↓
검증 체크리스트 출력
    ↓
완료 폴더에 사방넷 입력본 생성
    ↓
사람이 검증 후 사방넷 업로드
```

---

## 3. 운영 원칙

### 3-1. 실행 성공과 업로드 가능은 다르다

`완료 성공 N건 / 실패 0건`이 나와도 업로드 가능하다고 판단하지 않는다.

다음 문제가 숨어 있을 수 있다.

```text
새 거래처 식별 실패
새 모델 단가 미등록
단가 미매칭 후 기본단가 사용
모델명이 원본 상품명 일부로 남음
합계 행이 주문으로 처리됨
세트 상품이 단품으로 처리됨
택배비 기준 불일치
```

### 3-2. 금액은 내부 단가테이블 기준

거래처 발주서 원본의 공급단가, 공급가합, 택배비는 원칙적으로 사용하지 않는다.

```text
사방넷 입력본의 공급단가와 택배비는
거래처_단가테이블.xlsx의 내부 기준값으로 생성한다.
```

거래처 발주서와 내부 약정 단가가 다르면 원본 값을 따라가지 않는다.
내부 `거래처_단가테이블.xlsx`를 수정하여 기준을 바로잡는다.

### 3-3. 배송메시지는 가져온다

배송메시지는 고객 요청사항이므로 가능한 한 원본 발주서에서 가져온다.

```text
K_배송메시지 = 원본 배송메시지 컬럼 연결 권장
L_공급가합 = 원칙적으로 비움
M_공급단가 = 원칙적으로 비움
N_택배비 = 원칙적으로 비움
단가_원본에서(Y/N) = 원칙적으로 N
```

---

## 4. 시스템 경계

이 변환기는 매출관리포털 본체와 목적이 다르다.

| 구분 | 역할 |
|---|---|
| 사방넷 발주서 변환기 | 외부 거래처 발주서 → 사방넷 입력본 생성 |
| 매출관리포털 | 사방넷 원본 주문 이후 원장/출고/송장/ERP/마감 관리 |

따라서 이 문서 세트는 `sales-portal-*` 문서와 별도로 관리한다.


## 6. AI 에이전트 주의사항

AI 에이전트는 아래를 임의로 변경하지 않는다.

```text
단가_원본에서(Y/N)을 Y로 바꾸지 말 것
원본 공급단가를 기준으로 삼지 말 것
매핑테이블에 같은 거래처명을 여러 행으로 만들지 말 것
단가 미매칭을 실행 성공으로 판단하지 말 것
합계/총계 행을 주문으로 처리하지 말 것
PFW-901, 세트, 트레이 경고를 무시하지 말 것
```
