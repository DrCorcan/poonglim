# 사방넷 발주서 변환 검증 규칙

문서 버전: v1.0  
작성일: 2026-06-19

---

## 1. 목적

이 문서는 `sabang.py` 실행 후 사방넷 입력본을 업로드해도 되는지 판단하기 위한 검증 기준이다.

핵심 원칙:

```text
프로그램 실행 성공 여부가 아니라
업로드해도 안전한 출력인지 판단한다.
```

---

## 2. 검증 순서

### 2-1. 발견 파일 수 확인

```text
발견된 파일: N개
```

원본 폴더에 넣은 발주서 수와 일치해야 한다.

### 2-2. 거래처 식별 확인

정상:

```text
✅ 파일명.xlsx → 거래처명
```

오류:

```text
❌ 파일명.xlsx → 거래처 식별 불가
```

거래처 식별 불가 시 매핑테이블에 거래처 신규 행을 추가한다.

### 2-3. 변환 건수 확인

```text
거래처: 주니조아 / 1건
```

원본 실제 주문 건수와 변환 건수가 맞아야 한다.
합계/총계 행이 포함되면 안 된다.

### 2-4. 검증 체크리스트 확인

반드시 아래 항목을 본다.

```text
수취인
수량
모델명
공급단가
택배비
비고
주문번호
```

---

## 3. 업로드 가능 기준

아래 조건을 모두 만족해야 한다.

```text
거래처 식별 성공
변환 건수 정상
수취인이 실제 고객명
합계/총계 행 없음
모델명이 사방넷모델명 기준으로 정상 출력
공급단가가 내부 단가테이블 기준과 일치
택배비가 내부 단가테이블 기준과 일치
주문번호가 정상 생성 또는 원본 주문번호 사용
세트/트레이/합포장 경고 확인 완료
```

---

## 4. 업로드 금지 기준

아래 중 하나라도 있으면 업로드하지 않는다.

```text
거래처 식별 불가
모델명이 원본 상품명 일부로 출력됨
모델명 공란
공급단가가 예상과 다름
새 상품인데 기존 기본단가로 출력됨
합계/총계 행이 주문으로 들어감
수량이 공란 또는 이상함
PFW-901 세트 여부 미확인
세트 상품인데 단품으로 출력됨
원본 시트 선택이 의심됨
```

---

## 5. 기본단가 미매칭 의심 패턴

### 패턴 1. 모델명이 상품명 일부로 나옴

예:

```text
신혜순 / [윈세이] 24L 1등 / 10000 / 3000
```

판단:

```text
W24L1A 상품코드가 단가테이블에 없어 기본단가로 빠진 것일 수 있음
```

### 패턴 2. 모델명 공란

예:

```text
김태연 / 모델명 공란 / 28000 / 3000
```

판단:

```text
상품코드 미매칭 가능성 큼
```

### 패턴 3. 새 상품인데 첫 번째 단가가 반복됨

예:

```text
새 모델인데 CHC-UY01 단가 26000이 출력됨
```

판단:

```text
거래처 첫 번째 기본단가로 빠졌을 가능성
```

## 6. 경고 처리 기준

### 6-1. 당일 시트 자동 선택 실패

예:

```text
⚠️ 당일 시트 자동 선택 실패 → 첫 번째 시트 사용: Sheet1
```

판단 기준:

```text
첫 번째 시트가 실제 발주 시트이면 문제 없음
첫 번째 시트가 안내/양식/이전 날짜이면 문제
```

### 6-2. PFW-901 감지

예:

```text
⚠️ PFW-901 감지: 세트 여부 확인 필요
```

판단 기준:

```text
PFW-901 단품이면 업로드 가능
트레이 포함/세트/워머+트레이이면 분리 여부 확인 필요
```

### 6-3. 세트 상품 감지

예:

```text
⚠️ 세트 상품 감지: 사방넷 입력 시 2행 분리 필요
```

판단 기준:

```text
사방넷모델명에 +가 포함된 세트 모델은 행 분리 처리 필요
배송비는 첫 번째 행에만 부여하는지 확인
```

---

## 7. 택배비 검증 기준

일반 기준:

```text
단가테이블 택배비 사용
제주 주소는 제주추가택배비 적용
PMK-3000 일부 수량은 별도 택배비 조정 가능
수량이 많아도 자동 추가되지 않는 거래처가 있으므로 거래처 기준 확인
```

주의 사례:

```text
CHC-UY01 수량 12개인데 택배비 3000원 1회 출력
→ 거래처 기준이 맞으면 정상
→ 박스 수량별 배송비가 필요하면 수동 확인
```

## 8. 최종 판정 문구

검증 후 거래처별로 아래처럼 판단한다.

```text
업로드 가능
조건부 업로드 가능
업로드 금지
매핑테이블 수정 필요
단가테이블 수정 필요
원본 발주서 확인 필요
```
