문서 버전: v1.1  
최종 수정일: 2026-06-19

## 1. 문서 목적

이 문서는 `sabang.py`, `거래처_매핑테이블.xlsx`, `거래처_단가테이블.xlsx`, 운영 검증 기준의 변경 이력을 날짜별로 기록한다.

원칙:

```text
구체적인 거래처별 변경 이력은 이 문서에만 기록한다.
troubleshooting.md, mapping-table.md, price-table.md에는 같은 거래처 사례를 중복 기재하지 않는다.
실제 현재값의 기준은 항상 거래처_매핑테이블.xlsx와 거래처_단가테이블.xlsx이다.
```

## 2026-06-19 — 문서 중복 정리 원칙 확정

### 변경 내용

`sabang-converter` 문서 세트에서 거래처별 구체 사례가 여러 문서에 중복되어 있던 구조를 정리하기로 했다.

정리 원칙:

```text
changelog.md = 유일한 변경 이력
troubleshooting.md = 증상 패턴과 확인 방법만 기록
mapping-table.md = 매핑테이블 컬럼 설명만 기록
price-table.md = 단가테이블 컬럼 설명만 기록
```

### 이유

같은 거래처 사례가 여러 문서에 반복되면, 실제 테이블 수정 후 문서 간 값이 서로 달라질 위험이 있다.  
AI 에이전트가 오래된 예시값을 실제 기준으로 오해하지 않도록 문서 역할을 분리한다.

### 영향 범위

```text
sabang-converter-changelog.md
sabang-converter-troubleshooting.md
sabang-converter-mapping-table.md
sabang-converter-price-table.md
```

## 2026-06-19 — 문서 체계 분리

### 변경 내용

매출관리포털 본체 문서와 별도로 `sabang-converter` 문서 세트를 분리했다.

추가 문서:

```text
README.md
sabang-converter-overview.md
sabang-converter-data-architecture.md
sabang-converter-mapping-table.md
sabang-converter-price-table.md
sabang-converter-validation-rules.md

```

### 이유

반면 `sabang.py`는 외부 거래처 발주서를 사방넷 입력본으로 변환하는 전처리 시스템이다. 두 시스템을 같은 문서 안에 섞으면 AI 에이전트가 업무 경계를 혼동할 수 있다.

## 2026-06-19 — 금액 기준 원칙 확정

### 변경 내용

사방넷 입력본의 금액 기준을 내부 단가테이블 기준으로 확정했다.

확정 규칙:

```text
거래처 원본 발주서의 공급가합, 공급단가, 택배비는 원칙적으로 사용하지 않는다.
사방넷 입력본의 금액은 내부 거래처_단가테이블.xlsx 기준으로 생성한다.
거래처 발주서와 내부 기준 단가가 다르면, 원본값을 따라가지 않고 내부 단가테이블을 수정한다.
```

매핑테이블 운영 기준:

```text
K_배송메시지 = 원본에서 가져온다.
L_공급가합 = 원칙적으로 비운다.
M_공급단가 = 원칙적으로 비운다.
N_택배비 = 원칙적으로 비운다.
단가_원본에서(Y/N) = 원칙적으로 N.
```

### 이유

거래처 발주서의 금액이 잘못 기재될 수 있다. 원본 금액을 그대로 사용하면 사방넷 입력본도 잘못 생성될 수 있으므로, 내부 기준 테이블을 단일 기준으로 삼는다.

## 2026-06-19 — 이데컴퍼니 신규 등록

### 변경 내용

이데컴퍼니 발주서를 처리하기 위해 매핑테이블과 단가테이블에 신규 거래처 기준을 추가했다.

매핑테이블 변경:

```text
거래처명 = 이데컴퍼니
파일명키워드 = 이데컴퍼니
주문번호prefix = ED
데이터시작행 = 2
K_배송메시지 = J
단가_원본에서(Y/N) = N
P_주문번호 = I
```

단가테이블 변경:

```text
거래처명 = 이데컴퍼니
상품코드 = CHC-UY01
사방넷모델명 = CHC-UY01
공급단가 = 25500
택배비 = 3000
```

### 이유

파일명에 이데컴퍼니가 포함되어 있었지만, 기존 매핑테이블에 거래처 행이 없어 거래처 식별이 되지 않았다. 이후 단가테이블에도 이데컴퍼니 행이 없어 정상 단가 매칭이 되지 않았다.

## 2026-06-19 — 주니조아 합계 행 제외 및 OE-N2P 단가 추가

### 변경 내용

주니조아 발주서에서 합계 행이 주문으로 처리되는 문제를 방지하기 위해 매핑테이블 필터를 추가했다.

매핑테이블 변경:

```text
업체필터컬럼 = B
업체필터값 = 선불
```

운영 기준:

```text
주니조아는 매핑테이블에 2개 행을 쓰지 않는다.
주니조아 매핑행은 1개만 사용한다.
```

단가테이블 변경:

```text
거래처명 = 주니조아
상품코드 = OE-N2P
사방넷모델명 = OE-N2P
공급단가 = 10000
택배비 = 3000
```

### 이유

원본 발주서의 합계 행이 실제 주문처럼 변환되었고, 신규 상품이 단가테이블에 없어 기존 기본단가로 잘못 처리될 위험이 있었다.

---

## 2026-06-19 — 구문 W24L1A 단가 추가

### 변경 내용

구문 거래처의 W24L1A 상품 기준을 단가테이블에 추가했다.

단가테이블 변경:

```text
거래처명 = 구문
상품코드 = W24L1A
사방넷모델명 = W24L1A
공급단가 = 212500
택배비 = 4000
```

### 이유

신규 제습기 상품이 단가테이블에 없어 상품명 일부가 모델명처럼 출력되고, 기존 기본단가로 잘못 빠지는 문제가 있었다.

### 참고

구문 발주서는 시트명 경고가 발생할 수 있다. 실제 발주 데이터가 `당일발주` 시트에 있으면 치명 오류로 보지 않는다.

## 2026-06-19 — 크레이브 모기퇴치기 매칭 보완

### 변경 내용

크레이브 발주서의 모기퇴치기 상품이 내부 사방넷 모델명으로 변환되도록 단가테이블을 보완했다.

단가테이블 변경:

```text
거래처명 = 크레이브
상품코드 = 가정용 모기퇴치기 또는 PMK-3000
사방넷모델명 = PMK-3000
공급단가 = 크레이브 내부 기준 단가
택배비 = 3000
```

### 이유

기존 단가테이블에 해당 상품 키워드가 없어 상품명 일부가 모델명처럼 출력되고, 거래처의 기존 기본단가로 잘못 처리될 위험이 있었다.

## 2026-06-17~2026-06-18 — 냉감패드 단가 미매칭 보완

### 변경 내용

냉감패드 관련 신규 상품 코드가 거래처별 단가테이블에 추가되었다.

```text
상품코드 = PCN-2000
사방넷모델명 = PCN-2000
공급단가 = 30000
택배비 = 3000
```

에쓰앤씨:

```text
상품코드 = PCM-1000
사방넷모델명 = PCN-1000
공급단가 = 28000
택배비 = 3000
```

### 이유

신규 냉감패드 상품이 단가테이블에 없어 기존 기본단가로 잘못 처리될 위험이 있었다. 특히 모델명 변환이 필요한 상품은 사방넷모델명을 내부 기준으로 명확히 지정해야 한다.
