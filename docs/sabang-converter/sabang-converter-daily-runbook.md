# 사방넷 발주서 변환 일일 운영 절차

문서 버전: v1.0  
작성일: 2026-06-19

---

## 1. 목적

이 문서는 매일 들어오는 거래처 발주서를 사방넷 입력본으로 변환하고 검증하는 운영 절차를 정의한다.

---

## 2. 사전 준비

### 2-1. 원본 폴더 정리

원본 폴더에는 오늘 처리할 발주서만 둔다.

```text
E:\★ 풍림전자\경영관리\1. 사방넷변환 폴더\원본
```

이전 날짜 발주서가 섞이면 중복 생성 위험이 있다.

### 2-2. 완료 폴더 확인

완료 폴더에 이전에 잘못 생성된 파일이 있을 수 있으므로 업로드할 파일명을 주의한다.

```text
E:\★ 풍림전자\경영관리\1. 사방넷변환 폴더\완료
```

잘못 생성된 파일은 업로드하지 않도록 별도 구분한다.

---

## 3. 실행 전 확인

```text
sabang.py
거래처_매핑테이블.xlsx
거래처_단가테이블.xlsx
원본 발주서 파일들
```

`.xls` 파일이 있으면 PC에 `xlrd`가 설치되어 있어야 한다.

확인 명령:

```bash
python -c "import pandas, openpyxl, xlrd; print('OK')"
```

---

## 4. 실행

`sabang.py`를 실행한다.

정상 시작 로그 예:

```text
풍림전자 사방넷 입력본 자동 변환 v3.8
등록된 거래처: N개
단가테이블: N개 거래처
발견된 파일: N개
```

---

## 5. 1차 검증: 거래처 식별

먼저 발견된 파일 목록을 본다.

정상:

```text
✅ 파일명.xlsx → 거래처명
```

오류:

```text
❌ 파일명.xlsx → 거래처 식별 불가
```

거래처 식별 불가 시:

```text
거래처_매핑테이블.xlsx에 신규 거래처 행 추가
거래처_단가테이블.xlsx에도 해당 거래처 상품 추가
재실행
```

---

## 6. 2차 검증: 거래처별 체크리스트

각 거래처의 검증 체크리스트를 확인한다.

확인 항목:

```text
수취인
수량
모델명
공급단가
택배비
비고
주문번호
```

---

## 7. 3차 검증: 업로드 금지 패턴

아래 패턴이 있으면 업로드하지 않는다.

```text
합계 행이 들어감
모델명 공란
모델명이 상품명 일부로 출력됨
새 상품인데 기존 기본단가 출력
공급단가/택배비가 예상과 다름
거래처 식별 실패
세트 경고 미확인
```

---

## 8. 수정 절차

### 8-1. 거래처 식별 실패

```text
거래처_매핑테이블.xlsx 수정
파일명키워드 확인
데이터시작행 확인
컬럼 매핑 확인
```

### 8-2. 단가 미매칭

```text
거래처_단가테이블.xlsx 수정
상품코드 키워드 추가
사방넷모델명 추가
공급단가/택배비 추가
```

### 8-3. 합계 행 포함

```text
거래처_매핑테이블.xlsx의 업체필터컬럼/업체필터값 추가
```

예:

```text
주니조아
업체필터컬럼 = B
업체필터값 = 선불
```

## 9. 재실행

수정 후 다시 실행한다.

정상 결과가 나올 때까지 아래 과정을 반복한다.

```text
실행
검증 체크리스트 확인
문제 거래처 수정
재실행
```

## 10. 최종 업로드

업로드 가능한 파일만 사방넷에 업로드한다.

업로드하지 말아야 할 파일:

```text
이전 날짜에 잘못 생성된 파일
수정 전 생성 파일
업로드 금지 판정 거래처 파일
검증 전 파일
```
