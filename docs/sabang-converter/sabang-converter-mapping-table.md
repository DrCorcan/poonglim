# 거래처_매핑테이블 데이터 사전

문서 버전: v1.1  
최종 수정일: 2026-06-19

---

## 1. 문서 목적

`거래처_매핑테이블.xlsx`는 거래처별 발주서 양식을 표준 사방넷 입력본 구조로 해석하기 위한 기준 테이블이다.

거래처마다 발주서 컬럼 위치가 다르기 때문에, `sabang.py`는 이 테이블을 기준으로 원본 파일에서 날짜, 수취인, 주소, 상품명, 모델명, 수량, 배송메시지, 주문번호 등을 읽는다.

이 문서는 **컬럼 의미와 운영 원칙만 설명**한다.

```text
거래처별 실제 값은 거래처_매핑테이블.xlsx에만 둔다.
거래처별 예시값과 변경 이력은 이 문서에 중복 기재하지 않는다.
변경 이력은 sabang-converter-changelog.md를 확인한다.
```

---

## 2. 기본 원칙

```text
거래처 1개 = 매핑테이블 1행
```

같은 거래처명을 여러 행으로 만들지 않는다.  
현재 `sabang.py` 구조에서는 같은 거래처명이 여러 행이면 나중 행이 앞 행을 덮어쓸 수 있다.

거래처별 실제 컬럼 매핑값은 `거래처_매핑테이블.xlsx`가 기준이다.

## 3. 컬럼 정의

| 컬럼명 | 의미 | 필수 여부 | 운영 기준 |
|---|---|---|---|
| 거래처명 | 내부 거래처명 | 필수 | 단가테이블의 거래처명과 정확히 일치해야 한다. |
| 파일명키워드 | 파일명에서 거래처를 식별하는 키워드 | 필수 | 원본 파일명에 포함되어야 한다. |
| 주문번호prefix | 자동 주문번호 생성 prefix | 필수 | 원본 주문번호가 없을 때 사용한다. |
| 데이터시작행 | 실제 주문 데이터 시작 행 | 필수 | 엑셀 화면 기준 1부터 입력한다. |
| A_날짜 | 원본 발주서의 날짜 컬럼 | 권장 | 비어 있거나 값이 부정확하면 실행일 기준 주문번호가 생성될 수 있다. |
| B_수취인 | 원본 수취인 컬럼 | 필수 | 수취인이 비어 있으면 행 제외 기준이 될 수 있다. |
| C_우편번호 | 원본 우편번호 컬럼 | 선택 | 없으면 빈칸으로 출력한다. |
| D_주소 | 원본 주소 컬럼 | 필수 | 주소 누락은 사방넷 업로드 사고로 이어질 수 있다. |
| E_전화번호1 | 원본 연락처1 컬럼 | 권장 | 주 연락처로 사용한다. |
| F_전화번호2 | 원본 연락처2 컬럼 | 선택 | 보조 연락처로 사용한다. |
| G_상품명 | 원본 상품명 컬럼 | 필수 | 단가테이블 상품코드 매칭 검색텍스트에 포함된다. |
| H_모델명 | 원본 모델명 컬럼 | 권장 | 단가테이블 상품코드 매칭 검색텍스트에 포함된다. |
| I_수량 | 원본 수량 컬럼 | 필수 | 숫자 변환 실패 시 기본값으로 처리될 수 있으므로 검증이 필요하다. |
| K_배송메시지 | 원본 배송메시지 컬럼 | 권장 | 고객요청메시지로 출력한다. |
| L_공급가합 | 원본 공급가합 컬럼 | 원칙적으로 비움 | `단가_원본에서(Y/N)`이 N이면 사용하지 않는다. |
| M_공급단가 | 원본 공급단가 컬럼 | 원칙적으로 비움 | `단가_원본에서(Y/N)`이 N이면 사용하지 않는다. |
| N_택배비 | 원본 택배비 컬럼 | 원칙적으로 비움 | `단가_원본에서(Y/N)`이 N이면 사용하지 않는다. |
| 단가_원본에서(Y/N) | 원본 금액 사용 여부 | 필수 | 원칙적으로 N으로 둔다. |
| O_옵션 | 원본 옵션 컬럼 | 선택 | 상품코드 매칭 검색텍스트에 포함된다. 색상/규격 구분에 중요할 수 있다. |
| 업체필터컬럼 | 처리 대상 행 필터 컬럼 | 선택 | 합계/총계 행 제외, 특정 업체 행만 추출할 때 사용한다. |
| 업체필터값 | 처리 대상 행 필터 값 | 선택 | 원본 필터 컬럼 값과 정확히 일치하는 행만 처리한다. |
| P_주문번호 | 원본 주문번호 컬럼 | 선택 | 값이 있으면 발주서 주문번호를 우선 사용한다. |

## 4. 금액 컬럼 운영 원칙

아래 3개 컬럼은 특별한 경우가 아니면 비운다.

```text
L_공급가합
M_공급단가
N_택배비
```

이유:

```text
사방넷 입력본의 금액은 거래처 원본 발주서가 아니라 내부 거래처_단가테이블.xlsx 기준으로 생성한다.
거래처 발주서 금액이 잘못 들어와도 내부 기준을 따라야 한다.
```

따라서 `단가_원본에서(Y/N)`은 원칙적으로 `N`이다.

---

## 5. 배송메시지 운영 원칙

`K_배송메시지`는 금액 컬럼과 다르다.  
고객 요청사항이므로 가능한 한 원본 발주서의 배송메시지 컬럼을 연결한다.

```text
K_배송메시지 = 원본 배송메시지 컬럼 연결 권장
```

단, 거래처 발주서의 배송메시지 컬럼에 내부 메모나 불필요한 값이 섞이면 연결 여부를 별도로 판단한다.

---

## 6. 주문번호 운영 원칙

거래처 발주서에 주문번호가 있으면 그 주문번호를 우선 사용한다.

```text
P_주문번호에 연결된 원본 값이 있음 → 원본 주문번호 사용
P_주문번호가 비어 있음 또는 원본 값이 공란 → 주문번호prefix로 자동생성
```

실행 로그에서 원본 주문번호를 사용한 행은 `★` 표시로 구분된다.

검증 기준:

```text
원본 발주서에 주문번호가 있는데 자동생성 번호가 나오는지 확인한다.
자동생성이 정상인 거래처인지 확인한다.
```

## 7. 필터 컬럼 운영 원칙

합계 행, 총계 행, 빈 행, 특정 조건이 아닌 행을 제외해야 할 때 `업체필터컬럼`과 `업체필터값`을 사용한다.

작동 방식:

```text
원본 발주서의 업체필터컬럼 값이 업체필터값과 정확히 일치하는 행만 처리한다.
```

주의:

```text
필터는 원본 발주서의 컬럼 기준이다.
사방넷 출력 파일의 컬럼 기준이 아니다.
```

같은 거래처를 여러 매핑행으로 나누는 방식은 사용하지 않는다.  
거래처별 구체 필터 변경 이력은 `sabang-converter-changelog.md`를 확인한다.

---

## 8. 컬럼 문자 입력 규칙

엑셀 컬럼은 A, B, C처럼 알파벳으로 입력한다.

```text
A
B
C
J
T
```

비워도 되는 컬럼은 빈칸으로 둔다.  
사용하지 않는 금액 관련 컬럼은 임의로 원본 컬럼을 연결하지 않는다.

## 9. 수정 후 검증 기준

매핑테이블을 수정한 뒤에는 실행 로그에서 아래를 확인한다.

```text
거래처 식별 성공 여부
변환 건수와 실제 주문 건수 일치 여부
합계/총계 행 포함 여부
수취인 공란 행 포함 여부
주문번호가 원본 우선인지 자동생성인지
배송메시지 누락 여부
상품명/모델명/옵션이 단가 매칭에 충분히 들어오는지
```
