# 거래처_단가테이블 데이터 사전

문서 버전: v1.1  
최종 수정일: 2026-06-19

---

## 1. 문서 목적

`거래처_단가테이블.xlsx`는 거래처별 상품 매칭, 공급단가, 택배비, 사방넷모델명을 결정하는 내부 기준 테이블이다.

사방넷 입력본의 금액은 거래처 발주서 원본이 아니라 이 테이블을 기준으로 생성한다.

이 문서는 **컬럼 의미와 운영 원칙만 설명**한다.

```text
거래처별 실제 상품코드, 공급단가, 택배비, 사방넷모델명은 거래처_단가테이블.xlsx에만 둔다.
거래처별 구체 예시 표는 이 문서에 두지 않는다.
변경 이력은 sabang-converter-changelog.md를 확인한다.
```

---

## 2. 기본 원칙

```text
거래처별로 같은 상품이라도 단가가 다를 수 있으므로
거래처명 + 상품코드 조합으로 관리한다.
```

새 거래처 또는 새 모델이 들어오면 반드시 이 테이블에 추가한다.

실제 운영 기준:

```text
거래처 발주서 원본 금액보다 내부 단가테이블을 우선한다.
원본 금액과 내부 기준이 다르면 원본을 따라가지 않고 단가테이블을 수정한다.
```

## 3. 컬럼 정의

| 컬럼명 | 의미 | 실제 프로그램 사용 여부 | 운영 기준 |
|---|---|---|---|
| 거래처명 | 거래처 구분 | 사용 | 매핑테이블의 거래처명과 정확히 일치해야 한다. |
| 상품코드 | 원본 상품명/모델명/옵션에서 찾는 매칭 키워드 | 사용 | 너무 넓지 않게, 가능한 한 모델 단위로 입력한다. |
| 상품명 | 사람이 보기 위한 설명 | 현재 매칭에는 사용 안 함 | 검수 편의를 위한 설명값이다. |
| 공급단가 | 사방넷 입력본 공급단가 | 사용 | 내부 기준 단가를 입력한다. |
| 택배비 | 사방넷 입력본 택배비 | 사용 | 내부 기준 택배비를 입력한다. |
| 제주추가택배비 | 제주 주소일 때 추가 배송비 | 사용 | 주소에 제주가 포함될 때 추가될 수 있다. |
| 사방넷모델명 | 최종 출력 모델명 | 사용 | 사방넷 입력본의 모델명 칸에 출력된다. |

## 4. 상품코드, 상품명, 사방넷모델명의 차이

```text
상품코드 = 프로그램이 원본 텍스트에서 찾는 검색 키워드
상품명 = 사람이 보기 위한 설명
사방넷모델명 = 사방넷 입력본에 최종 출력할 모델명
```

주의:

```text
상품명은 현재 sabang.py의 매칭 기준으로 직접 사용되지 않는다.
실제 매칭은 상품코드 기준이다.
```

따라서 단가테이블 관리에서 가장 중요한 값은 `상품코드`와 `사방넷모델명`이다.

---

## 5. 상품 매칭 방식

프로그램은 원본 발주서 행에서 아래 텍스트를 합쳐 검색한다.

```text
검색텍스트 = 원본 모델명 + 원본 상품명 + 원본 옵션
```

그리고 단가테이블의 `상품코드`가 검색텍스트 안에 포함되면 해당 상품으로 매칭한다.

운영상 의미:

```text
원본 상품명에 모델코드가 들어 있으면 상품코드에 모델코드를 넣는 것이 안전하다.
원본 모델명 컬럼이 따로 있으면 H_모델명 매핑이 중요하다.
색상/규격이 옵션에만 있으면 O_옵션 매핑이 중요하다.
```

---

## 6. 상품코드 작성 원칙

상품코드는 가능한 한 구체적으로 작성한다.

좋은 방향:

```text
모델코드
거래처 발주서에 반복적으로 들어오는 고유 상품 키워드
사방넷모델명으로 변환 가능한 좁은 키워드
```

위험한 방향:

```text
너무 넓은 일반명사
여러 상품에 공통으로 들어가는 단어
규격 구분 없이 싱글/더블처럼 혼동될 수 있는 단어
```

단, 원본 발주서에 모델코드가 전혀 없고 상품명만 반복되는 거래처는 상품명 키워드를 사용할 수 있다. 이 경우에도 가능한 한 거래처별로 좁은 키워드를 사용한다.

## 7. 내부 단가 기준 원칙

거래처 원본 발주서의 단가가 내부 단가와 다르면 원본 값을 따라가지 않는다.

```text
원본 발주서 단가 오류 가능성 있음
→ 원본값 사용 금지
→ 내부 거래처_단가테이블.xlsx 수정
→ 재실행
```

매핑테이블에서도 특별한 경우가 아니면 아래 원칙을 따른다.

```text
단가_원본에서(Y/N) = N
L_공급가합 = 비움
M_공급단가 = 비움
N_택배비 = 비움
```

## 8. 기본단가 오사용 위험

현재 프로그램 구조상 상품코드 매칭에 실패하면 거래처의 첫 번째 단가가 기본단가처럼 사용될 수 있다.

이 경우 실행은 성공하지만 출력은 틀릴 수 있다.

의심 증상:

```text
새 상품인데 기존 상품 단가가 나옴
모델명이 사방넷모델명으로 변환되지 않음
상품명 일부가 모델명 칸에 출력됨
공급단가가 해당 거래처의 첫 번째 상품 단가와 같음
```

검증 방법:

```text
실행 로그의 모델명, 공급단가, 택배비를 단가테이블 기준과 비교한다.
새 상품이면 반드시 단가테이블에 상품코드가 있는지 확인한다.
```

---

## 9. 택배비 운영 원칙

`택배비`는 단가테이블의 내부 기준값을 사용한다.

추가 확인이 필요한 경우:

```text
수량이 많은 주문
제주 주소
합포장
상품별 수량별 택배비 예외가 있는 경우
거래처별 특수 배송비 기준이 있는 경우
```

택배비 예외가 반복되면 단가테이블 또는 코드 로직에 별도 규칙을 추가한다.

## 10. 수정 후 검증 기준

단가테이블을 수정한 뒤에는 실행 로그에서 아래를 확인한다.

```text
모델명이 사방넷모델명으로 출력되는가
공급단가가 내부 기준 단가와 일치하는가
택배비가 내부 기준과 일치하는가
동일 거래처 다른 상품 단가로 잘못 빠지지 않았는가
새 상품이 기존 기본단가로 처리되지 않았는가
```

거래처별 구체 변경 내역은 `sabang-converter-changelog.md`를 확인한다.
