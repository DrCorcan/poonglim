## 1. 이 단계가 하는 일

캐리어가 매달 보내는 **운영안** 엑셀을 읽어서, 모델별 단가를 구글시트
`캐리어_단가이력`에 누적 기록하고, 가장 최근 달 기준으로
`캐리어_모델단가테이블.xlsx`의 단가도 자동 갱신한다.

```
캐리어 → 운영안 파일(26년_N월_*.xlsx) 매달 발송 (수동 다운로드)
        ↓
원본 폴더에 넣고 캐리어단가이력업로드.py 실행
        ↓
운영안에서 "벤더 공통" 시트 파싱
        ↓
구글시트 캐리어_단가이력에 추가 (같은 연월 있으면 덮어쓰기 확인)
        ↓
최신 연월 기준으로 캐리어_모델단가테이블.xlsx 일부 컬럼 갱신
        ↓
처리된 운영안 파일 → 완료\YYYYMM\ 로 이동
```

월 1회 실행하는 프로그램이다 (캐리어가 운영안을 보낼 때마다).

## 2. 운영안 파일에서 시트 인식 방법

```python
시트명후보 = ['벤더 공통', '벤더 공통 (2)', '벤더공통']
```
이 후보 중 하나가 있으면 그 시트를 쓰고, 없으면 시트 이름에 `'벤더'`가 포함된
첫 번째 시트를 사용한다. 둘 다 없으면 파싱을 포기하고 건너뛴다.

### ⚠️ 홈쇼핑 시트는 이 단계가 전혀 읽지 않는다
운영안 파일에는 `홈쇼핑` 시트가 따로 있는데(TARB-0062GDWSD, EARB-0102GDWSD 단가),
이 프로그램은 `시트명후보`에 홈쇼핑이 없어 해당 시트를 아예 시도하지 않는다.
01_매입처발주.md·02_매입처발주서생성.md의 "할 일" 목록에 이미 같은 항목이 있던
부분이며, 아직 해결되지 않은 상태다 (10번 항목 참조).

## 3. 연월 추출 방법

```text
1순위: 파일명에서 추출
       정규식 (\d{2})년[_\s]*(\d{1,2})월   예: "26년_6월" → 2026-06

2순위: 1순위 실패 시 — 파일 내용 첫 5행을 훑어서 추출
       정규식 (\d{4})년\s*(\d{1,2})월

둘 다 실패하면 해당 파일은 파싱하지 않고 건너뛴다 (⚠️ 연월 추출 실패 출력)
```

## 4. 모델 행 판별 및 컬럼 추출

```python
모델패턴 = re.compile(r'^[A-Z]{2,}[-]?[A-Z0-9]{3,}')
```
대시(`-`)가 없는 모델코드(예: 일부 구형 모델)도 매칭되도록 대시를 선택적으로 처리한다.
이 정규식에 안 맞는 행(소계/안내문 등)은 자동으로 건너뛴다.

컬럼은 정확한 이름이 아니라 **부분 문자열 포함 여부**로 찾는다 (운영안 파일마다 헤더
표기가 살짝 다를 수 있어서 유연하게 잡아둔 것으로 보인다):

| 찾는 컬럼 | 판단 기준(포함 문자열) |
|---|---|
| 모델컬럼 | `'모델'` |
| 공급가_포함컬럼 | `'설치비포함'` |
| 공급가_제외컬럼 | `'설치비제외'` 포함 AND `'자재'` 미포함 (자재비 컬럼과 혼동 방지) |
| 판매가컬럼 | `'판매가격'` |
| 카테고리컬럼 | `'카테고리'` |
| 평형컬럼 | `'평형'` |

모델/공급가포함/공급가제외 세 개 중 하나라도 못 찾으면 그 파일은 파싱을 포기한다.

카테고리 컬럼은 병합 셀 대응을 위해 forward-fill(`ffill()`) 처리한다 — 운영안에서
카테고리명이 첫 행에만 적혀있고 아래로 병합되어 있는 구조이기 때문이다.

## 5. ⚠️ 판매가격 NaN 필터 없음 — 알려진 위험

02_매입처발주서생성.md 8-2/8-4번 항목에 이미 기록된 사안과 같은 문제다.

운영안의 "벤더 공통" 시트에는 정상 단가 섹션과 "설치현장 특판건" 섹션이 같은
모델코드로 중복 존재할 수 있고, 특판건 행은 `판매가격` 컬럼이 빈 값(NaN)으로
구분된다 — 이게 2026-06-19에 발견된 EASB 6개 모델 단가 오류의 원인이었다.

**이 코드는 그 필터가 없다.** `parse_운영안()` 함수는 `모델패턴`에 맞는 행이면
판매가격이 비어있어도 그대로 추출한다:

```python
def num(v):
    try: return round(float(v))
    except: return ''
...
rows.append([
    연월, ..., 모델_clean, 모델,
    num(row.get(판매가컬럼, '')) if 판매가컬럼 else '',   # ← NaN이어도 그냥 통과
    num(row.get(공급가_포함컬럼, '')),
    num(row.get(공급가_제외컬럼, '')),
    등록일시,
])
```
즉 특판건 행도 정상 행과 똑같이 `캐리어_단가이력` 시트에 들어간다. 현재
`캐리어_모델단가테이블.xlsx`는 (이번 작업으로) 수동 재구축되어 정상화된 상태지만,
**다음 달 운영안이 들어와서 이 프로그램을 그대로 실행하면 같은 중복 문제가 재발할
수 있다.** (10번 할 일 목록 참조)

## 6. 캐리어_단가이력 시트 업데이트 동작

```text
'캐리어_단가이력' 시트가 없으면 → 새로 생성 (헤더 자동 입력)
같은 연월 데이터가 이미 있으면 → "덮어쓰시겠습니까? (y/n)" 콘솔 확인
  y 선택 시: 기존 해당 연월 행을 전부 찾아 삭제 후 새로 추가
  n 선택 시: 그 파일은 건너뜀
신규 연월이면 → append_rows로 바로 추가
```
이 확인 절차 때문에 이 프로그램은 **콘솔에서 사람이 입력을 기다려야 하는 대화형
프로그램**이다 — bat 파일로 실행해도 중간에 멈춰서 y/n을 기다린다.

## 7. 캐리어_모델단가테이블.xlsx 자동 갱신 로직 — ⚠️ 확인 필요

```python
최신연월 = df_이력['연월'].max()
df_최신 = df_이력[df_이력['연월'] == 최신연월].copy()
df_최신 = df_최신.drop_duplicates(subset=['모델코드'], keep='first')   # 모델코드만으로 중복 제거

for _, 이력행 in df_최신.iterrows():
    mask = df_단가['모델코드'].astype(str).str.strip() == 이력행['모델코드']
    if mask.any():
        if '공급가_VAT제외' in df_단가.columns:
            df_단가.loc[mask, '공급가_VAT제외'] = 이력행['공급가_설치포함_VAT제외']
        if '공급가_설치제외' in df_단가.columns:
            df_단가.loc[mask, '공급가_설치제외'] = 이력행['공급가_설치제외_VAT제외']
```

두 가지 확인이 필요한 지점을 발견했다 (코드만 읽고 판단한 것이라, 실제 파일을 열어서
확인이 필요하다 — 단정하지 않고 적어둔다):

**(1) 컬럼명 불일치 가능성.** 02_매입처발주서생성.md 8-3번 항목에 적힌 현재
`캐리어_모델단가테이블.xlsx`의 실제 컬럼명은 `공급가_설치포함 / 공급가_설치제외`다.
이 코드는 `'공급가_VAT제외'`라는 이름의 컬럼이 있는지 확인하는데, 그 이름이 실제
테이블에 없다면 `공급가_설치포함` 쪽은 갱신되지 않고 조용히 건너뛰어진다
(에러 없이 그냥 안 바뀜). `공급가_설치제외`는 이름이 정확히 일치하므로 갱신될 것으로
보인다.

**(2) 채널(벤더공통/홈쇼핑) 구분 없이 모델코드만으로 매칭.** 이 코드는 `모델코드`
컬럼만으로 단가테이블 행을 찾는다. 그런데 현재 단가테이블에는 TARB-0062GDWSD와
EARB-0102GDWSD가 **벤더공통 행과 홈쇼핑 행 두 개**로 존재한다(02_매입처발주서생성.md
8-3번 항목). 이 코드를 그대로 실행하면 `mask`가 두 행 모두에 걸리고, 두 행 모두
이 단계(벤더 공통 시트만 파싱)에서 가져온 같은 값으로 덮어써질 것으로 보인다 —
**홈쇼핑 단가 행이 벤더공통 값으로 잘못 덮어써질 위험.** 5번 항목의 홈쇼핑 시트
미파싱 문제와 함께 해결해야 한다.
