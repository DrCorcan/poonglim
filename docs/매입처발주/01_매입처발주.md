## 3. 사방넷주문번호 중복 입력 방지 (v1.9.2)

### 발견 경위

2026-06-30, Dr. Corcan이 캐리어_발주원장에서 같은 사방넷주문번호(2126749374, 강주현님)가 두 줄로 들어가 있는 걸 발견. 한 줄은 '발주완료'(오더번호 있음), 다른 한 줄은 '발주중'(오더번호 없음) — 같은 날(2026-06-26) 발주총괄을 두 번 실행하면서 생긴 것으로 추정.

### 추정 원인

기존 중복 방지 로직(`if 주문번호 and 주문번호 in 기존_캐리어: continue`)은 정상이지만, 사방넷주문번호 컬럼이 **엑셀에서 실행 회차에 따라 텍스트 또는 숫자(float)로 다르게 읽힐 수 있다.** 그러면:

```text
시트에 이미 저장된 값:  "2126749374"
이번에 새로 읽은 값:    "2126749374.0"   ← float로 읽혀서 .0이 붙음
```

사람 눈엔 같지만 파이썬은 다른 문자열로 보고, `in` 비교가 실패해서 중복 방지를 그냥 통과해버린다.

### 대응 (v1.9.2)

```text
safe_id(v) 함수 신규 추가
  → ID값 끝의 '.0'을 제거해 비교 전에 정규화
  → 캐리어/윈세이/스타리온 3곳 모두 사방넷주문번호 추출 시 safe()를 safe_id()로 교체

dedupe_rows_by_order_no(rows, cols, label) 함수 신규 추가
  → 구글시트에 append_rows 하기 직전, 이번 실행분 안에 같은 사방넷주문번호가
    또 있으면 첫 번째 행만 남기고 나머지 제외
  → 정상 동작이면 아무것도 안 걸러져야 함. 걸러지면 콘솔에 경고로 모델/번호 표시
    → 다른 원인이 또 있다는 신호이므로 확인 필요
  → 캐리어/윈세이/스타리온 3곳 모두 append_rows 직전에 적용
```

기존 로직(`기존_캐리어` 비교, `get_existing_map`)은 그대로 두고, 비교 직전 정규화 단계와 시트 업로드 직전 최종 필터만 추가한 것이라 기존 동작에 위험을 주지 않음.

### 한계

이미 잘못 들어간 과거 데이터(강주현님 행)는 이 코드로 자동 정리되지 않는다. 발견 시점 기준 구글시트에서 '발주중'(오더번호 없는 쪽) 행을 수동으로 삭제 처리함.

## 4. 마감단가 4종 — 매입정산 vs 발주서 단가의 구분 (v1.9 핵심)

### 배경

캐리어 전산은 물품대(설치비제외, 월별 변동)와 설치비(연간 고정)를 분리해서 관리한다. 발주서를 만들 때는 설치점이라도 Z01(설치비포함)/Z02(설치비미포함) 분기로 캐리어가 알아서 처리하므로, **발주서 자체는 의도적으로 설치비제외 단가(`캐리어공급가_VAT제외`)를 쓴다 — 이건 맞는 동작이고 바뀌지 않았다.**

문제는 매입정산(캐리어와의 마감 대조)이었다. 설치 건은 실제로는 설치비까지 포함된 금액으로 마감되어야 하는데, 이 흐름이 구글시트에 전혀 없었다. (2026-06-30 Dr. Corcan 보고로 발견)

### 계산 로직

```text
모델코드에 '-설치비제외' 접미사 있음 (사방넷 발주총괄 원본에서부터 붙어옴)
  → 설치점 배송이어도 무조건 배송상품(설치비 없음) 취급

모델코드에 '-할인점' 접미사 있음 (사방넷 발주총괄 원본에서부터 붙어옴)
  → 설치비와 무관, 채널만 무조건 벤더공통으로 강제 (홈쇼핑 전용 모델 판정 무시)

단가 매칭 시 위 두 접미사는 모두 떼고 모델코드_단가조회용으로 조회한다.

설치건여부 = (배송구분 == '설치점') AND (설치비제외 접미사 없음)

마감단가_VAT제외 = 설치건여부 ? 공급가_설치포함 : 공급가_설치제외
설치비_VAT제외   = 공급가_설치포함 - 공급가_설치제외   (검증용, 모델별 고정값)
마감금액_VAT제외 = round(마감단가_VAT제외 × 수량)
마감금액_VAT포함 = round(마감단가_VAT제외 × 수량 × 1.1)   ← 합산 후 마지막에 한 번만 ×1.1
```

물품대와 설치비를 각각 따로 VAT 붙여서 더하지 않는다 — 반드시 합산 먼저, VAT는 마지막에 한 번만.

### 부수 효과 — 버그 해소

접미사(`-설치비제외`, `-할인점`)가 붙은 모델코드는 v1.8까지 단가표 매칭에 실패해서 `캐리어공급가_VAT제외`가 빈칸으로 들어가고 있었다. v1.9.1에서 접미사를 떼고 매칭하도록 고치면서 이 문제도 함께 해소됨.

## 8. suppliers.html 현황판 — 마감금액 카드 (2026-06-30, 같은 날 1~4차 수정)

### 1차 수정 (당일 오전)

`ops.corcan.co.kr/operations/suppliers.html`의 "💰 이번달 발주 현황" 단 아래에 신규 섹션 추가:

```text
🧾 이번달 매입정산 마감금액 — 설치비포함, 캐리어 마감 대조용
  📦 이번달 마감건수
  💰 이번달 마감금액 (VAT별도 / VAT포함 두 줄 동시 표시)
```

계산은 화면(JS)에서 `allOrders`에 이미 포함된 `마감금액_VAT제외`/`마감금액_VAT포함`을 직접 합산하는 방식 — 서버 Apps Script(`getCarrierSummary`)는 건드리지 않음 (서버 코드 백업이 없어 안전하게 우회).

### 2차 수정 (당일 오후, Dr. Corcan 실사용 피드백 반영)

실제 화면을 본 뒤 세 가지 문제가 발견되어 수정함.

**문제 1 — 발주현황 목록(탭2) 금액이 여전히 설치비제외**
목록 행의 금액 컬럼이 `발주금액_VAT포함`(발주서 기준)을 그대로 쓰고 있었음. → **`마감금액_VAT포함`(매입정산 기준)으로 변경.** 마감금액이 없는 행(백필 전이거나 단가매칭 실패)만 "(발주서기준)" 표시를 달아 발주금액을 대신 표시.

**문제 2 — "이번달 발주금액" 카드가 오히려 혼란**
설치비제외 금액인데 "발주금액"이라고만 표시돼 있어 마감금액 카드와 헷갈림. → **카드 자체를 제거.** 3단은 "이번달 발주건수" / "전체 주문건수" 2개 카드로 단순화.

**문제 3 — 발주건수(37) vs 마감건수(40) 불일치**
1차 수정 때 마감건수/마감금액 집계를 `완료여부 IN (발주완료, 기존재고)`로 잡았던 게 원인. **'기존재고'는 이번 달에 캐리어로부터 새로 청구되는 금액이 아니다** — 예전에 매입해서 설치점에 미리 쌓아둔 재고를 꺼내 쓰는 것뿐이고, 그 매입대금은 처음 매입한 시점에 이미 정산이 끝난 상태. → **마감건수/마감금액 집계 기준을 '발주완료'만으로 축소.** 이후 마감건수(37) = 발주건수(37)로 정확히 일치.

```text
집계 기준 최종본:
  접수일이 이번달 AND 완료여부 = '발주완료'   ← 기존재고 제외 확정
```

기존 "이번달 발주금액" 카드(설치비제외, 발주서 기준)는 완전히 제거됨 — 위 문서 초판(1차 수정 시점) 설명과 달라진 부분이므로 혼동 주의.

발주현황 탭 개별 건 펼침 화면에도, 설치 건이면 마감금액(VAT+) 한 줄이 추가로 표시됨 (변동 없음).

### 3차 수정 (당일 오후 늦게, Dr. Corcan 추가 피드백 반영)

**문제 1 — 현황판에서 마감금액보다 발주건수가 먼저 보임**
"이번달 발주 현황"(건수만) 섹션이 "이번달 매입정산 마감금액" 섹션보다 위에 있어서, 더 중요한 정보인 마감금액이 스크롤해야 보이는 구조였음. → **두 섹션 순서를 맞바꿈.** 이제 화면 위에서부터: 1단(캐리어 발주 현황) → 2단(오늘 접수) → 3단(이번달 매입정산 마감금액, 먼저 노출) → 3-2단(이번달 발주 현황, 건수만) → 4단(취소·CS).

**문제 2 — 발주현황 목록(탭2)에 여전히 VAT포함 금액 하나만 표시**
2차 수정에서 목록 금액을 마감금액 기준으로 바꿨지만 VAT포함 한 값만 보여주고 있었음. → **VAT제외/VAT포함 두 값을 한 행에 병기.** `VAT- ₩... VAT+ ₩...` 형식. 마감금액이 없는 행(백필 전/단가매칭 실패)은 기존처럼 "(발주서기준)" 단일 값 표시 유지.

```text
최종 화면 구조 (2026-06-30 3차 수정 기준):
  1단 캐리어 에어컨 발주 현황 (6카드)
  2단 오늘의 주문 접수 현황 (5카드)
  3단 이번달 매입정산 마감금액 (마감건수 + VAT별도/포함, 가장 먼저 보이는 금액 정보)
  3-2단 이번달 발주 현황 (발주건수 + 전체주문건수, 건수만)
  4단 취소 및 CS 내역 (3카드)

발주현황 목록 행 금액 표시:
  마감금액 있음 → VAT- {금액}  VAT+ {금액}  (둘 다 표시)
  마감금액 없음 → {발주서기준 VAT+ 금액} (발주서기준)
```

### 4차 수정 (당일 저녁, Dr. Corcan 추가 피드백 반영)

**문제 — 발주현황 목록(탭2)에 헤더 행과 합계 행이 없음**
컬럼이 뭔지 한눈에 안 보이고, 조회 기간 전체 합계도 눈으로 일일이 더해야 했음. → **목록 상단에 컬럼 헤더 행 추가, 하단에 합계 행 추가.**

```text
하단 합계 행 집계 기준:
  마감금액(매입정산 기준)이 있는 행만 VAT-/VAT+ 각각 합산.
  마감금액 없는 행(백필 전/단가매칭 실패 구형모델 등)은 발주서기준 금액이라
  성격이 달라 합계에 섞지 않고, "N건 제외"로 별도 표기만 함.
```

이로써 같은 날(2026-06-30) suppliers.html에 1~4차까지 총 네 차례 수정이 있었음 — 코드(매입처발주.py)는 변경 없이 화면(JS)만 계속 다듬은 하루였다.
