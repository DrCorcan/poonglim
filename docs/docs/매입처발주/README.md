# 매입처 발주 자동화 문서 세트

이 문서 세트는 풍림전자 에어컨/제습기 매입처(캐리어·윈세이·스타리온) 발주 자동화 업무를
AI 에이전트와 안전하게 협업하기 위한 기준 문서다.

원칙: 각 문서는 해당 단계 프로그램의 **현재 동작 기준**만 담는다. 과거 버전 이력은 보관하지
않는다. 코드를 수정하면 그 즉시 해당 문서도 함께 수정한다.

## 문서 목록

| 파일 | 역할 | 최신 기준 |
|---|---|---|
| `00_캐리어운영안.md` | 0단계 — 캐리어단가이력업로드.py. 캐리어 운영안(매달 수신) → 단가테이블/이력 관리 | 미작성 |
| `01_매입처발주.md` | 1단계 — 매입처발주.py. 발주총괄 → 캐리어/윈세이/스타리온 구글시트 분기 업로드 | v1.6 |
| `02_매입처발주서생성.md` | 2단계 — 발주서생성.py. 구글시트 → 매입처별 발주서 생성 + 발송 | v2.5 |
| `03_발주원장업데이트.md` | 3단계 — 캐리어오더번호업데이트.py / 윈세이송장업데이트.py | 미작성 |

## 전체 흐름

```text
사방넷 발주총괄 다운로드 (수동)
        ↓
0단계 — 캐리어단가이력업로드.py   (매달 1회, 캐리어 운영안 → 단가테이블/이력)
        ↓
1단계 — 매입처발주.py             → 구글시트 캐리어/윈세이/스타리온_발주원장
        ↓
2단계 — 발주서생성.py             → 매입처별 발주서 생성 + (수동) 이메일 발송
        ↓
3단계 — 오더번호/송장 업데이트     → 구글시트 발주원장 완료여부 갱신
        ↓
ops.corcan.co.kr/operations/suppliers.html   ← 전체 현황 조회 및 수동 처리
```

## 핵심 주의 — 자주 발생하는 문제

```text
단가가 틀려 보인다
→ 코드보다 먼저 캐리어_모델단가테이블.xlsx(또는 0단계가 만드는 단가/이력)의
  중복·결측을 의심한다. 2026-06-19 운영안 "특판건" 섹션 혼입 중복 사례 참조
  (02_매입처발주서생성.md 8-2번 항목).

캐리어_모델단가테이블.xlsx는 1단계와 2단계가 공유한다
→ 파일 하나만 교체하면 양쪽 모두 자동 정상화된다. 코드는 건드릴 필요 없다
  (01_매입처발주.md 4-4번, 02_매입처발주서생성.md 8-5번 항목).

윈세이/스타리온 단가가 틀려 보인다
→ 2단계는 윈세이/스타리온 단가를 다시 조회하지 않는다. 1단계(매입처발주.py)와
  해당 _모델단가테이블.xlsx 쪽을 먼저 확인한다
  (02_매입처발주서생성.md 9-4번, 10-3번 항목).

캐리어 발주가 안 됐는데 신규 카운트에 안 보인다
→ 신규주문/주문확인 상태는 1단계에서 해피콜대기로 분류되고, 2단계는
  완료여부=신규 인 건만 본다. 해피콜대기는 정상적으로 제외된 것이다
  (01_매입처발주.md 3-2번, 02_매입처발주서생성.md 10-4번 항목).
```

## 재고 영향 있는 처리는 자동화하지 않는다

주문취소/취소재고 분기(오더번호 유무로 재고 복구 여부가 갈리는 처리)는 사람 확인 또는
명확한 기준으로만 처리한다. suppliers.html 취소처리 탭에서 안내 모달을 거쳐 처리한다.
