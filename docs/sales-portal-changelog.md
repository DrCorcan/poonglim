## 2026-06-17 — 매출관리포털 1차 설계 확정



```text

02_주문상세원장
오성 송장 매칭
```

## 2026-06-17 — 비오성 물류처 직행 흐름 추가

변경 내용:

```text
물류처가 오성물류가 아닌 주문은 최초 원장 적재 시 창고업로드상태=제외로 세팅한다.
해당 주문은 오성 업로드 파일 생성, 오성 업로드 완료 처리, 오성 송장자료 업로드 단계를 건너뛰고 송장/사방넷/ERP 단계로 직행한다.
```

변경 이유:

```text
쿠팡, 자체출고 등 비오성 주문이 오성 프로세스에 잘못 포함되는 것을 방지하기 위함.
```

영향 범위:

```text
02_주문상세원장
1번 오성 업로드 파일 생성
2번 오성 업로드 완료 처리

4번 사방넷 송장업로드 파일 생성
```

## 2026-06-17 — 오성 발주 후 고객 취소 제동 장치 추가

변경 내용:

```text
물류처=오성물류이고 창고업로드상태가 파일생성완료 또는 업로드완료인 상태에서 주문상태가 취소/반품/교환 계열로 바뀌면 오류상태=확인필요, 오류유형=출고전취소, 제동여부=Y로 자동 전환한다.
```

```text

```

```text
02_주문상세원장
91_오류로그
사방넷 송장업로드 파일 생성
거래처별 송장파일 생성
ERP 업로드 파일 생성
마감대조 실행
```

## 2026-06-17 — 거래처 송장 회신 기준 정정

변경 내용:

```text
거래처별 송장파일 생성 기준은 매입처가 아니라 점포명으로 확정했다.
점포명 → 거래처_히스토리 → 거래처명/이메일 매핑 구조를 사용한다.
```

변경 이유:

```text
송장을 회신해야 하는 대상은 상품 매입처가 아니라 판매 거래처/점포명이기 때문이다.
```

```text

6번 거래처별 송장파일 생성
7번 거래처 메일 발송 완료 처리
```

## 2026-06-17 — ERP 로직 보류 처리

변경 내용:

```text
ERP 업로드는 현재 기존 주문등록 업로드 로직을 임시 기준으로 유지하되, 향후 ERP 화면을 직접 보고 출고/반품/교환 중심 구조로 재설계할 가능성을 열어두었다.
```

변경 이유:

```text
ERP에 주문 전체를 넣는 것보다 실제 재고에 영향을 주는 출고, 반품, 교환 데이터를 넣는 것이 더 적절할 수 있기 때문이다.
```

영향 범위:

```text
07_ERP업로드이력
8번 ERP 업로드 파일 생성
9번 ERP 업로드 완료 처리
10번 마감대조 실행
```

확정 여부:

```text
임시 확정 / 향후 재검토
```
