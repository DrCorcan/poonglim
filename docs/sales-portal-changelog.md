# 풍림전자 매출관리포털 변경이력

이 문서는 풍림전자 매출관리포털 설계와 운영 규칙의 변경 사항을 누적 기록한다.

---


## 2026-06-19 — v1.2 마스터 드롭다운 적용 방침

### 1. 92/93 마스터 탭 드롭다운 우선 적용

결정 내용:

```text
92_품목마스터 / 93_거래처마스터 드롭다운을 우선 적용한다.
```

결정 이유:

```text
다음 단계가 오성 상품코드와 거래처 히스토리 마스터 데이터 입력이므로,
마스터 데이터 입력 전에 사용여부, 물류처, 메일발송여부 등 주요 선택값의 오타를 방지하기 위함이다.
```

실행 대상 함수:

```text
applyDropdowns92ItemMaster
applyDropdowns93CustomerMaster
```

적용 보류 대상:

```text
01_사방넷원본수집
02_주문상세원장
03_창고업로드이력
04_송장수신이력
05_사방넷송장업로드이력
06_거래처회신이력
07_ERP업로드이력
08_마감대조원장
90_실행로그
91_오류로그
```

보류 기준:

```text
위 나머지 탭들은 현재 데이터가 없으므로 즉시 적용하지 않는다.
사방넷 원본을 처음 실제로 흘려보내기 직전에 한 번에 적용한다.
```

영향 범위:

```text
92_품목마스터
93_거래처마스터
마스터 데이터 입력 절차
드롭다운 적용 순서
```

## 2026-06-18 — v1.1 설계 보완

### 1. 마스터 참조 탭 공식 추가

추가 탭:

```text
92_품목마스터
93_거래처마스터
```

추가 이유:

```text
오성 상품코드/바코드 매핑과 거래처 송장 회신 이메일 매핑의 기준 데이터 위치가 기존 설계에 명확히 정의되어 있지 않았다.
AI 에이전트와 Apps Script가 외부 엑셀 파일명을 추정하지 않고 구글시트 내부 마스터 탭을 기준으로 처리하도록 공식화했다.
```

영향 범위:

```text
오성 업로드 파일 생성
품목/모델코드 표준화
NS-3000 합포장/분리 판단
거래처별 송장파일 생성
거래처 이메일 매핑
```

### 2. 사방넷 원본 적재 Upsert 보존 규칙 추가

추가 이유:

```text
사방넷 원본을 매일 업로드할 때 기존 원장 행을 통째로 덮어쓰면, 이미 처리된 택배송장번호, 창고업로드상태, 사방넷송장업로드상태, ERP업로드상태, 오류/제동 정보가 초기화될 위험이 있다.
```

확정 규칙:

```text
사방넷주문번호가 없으면 신규 행 생성
사방넷주문번호가 있으면 기존 행 업데이트
단, 포털 내부 운영 상태값과 사람이 입력하거나 시스템이 처리한 값은 절대 덮어쓰지 않는다.
```

주문상태 변경 규칙:

```text
기존 현재주문상태 → 이전주문상태
신규 사방넷 주문상태 → 현재주문상태
변경 시 주문상태변경일시 기록
상태가 같으면 변경일시는 유지
```

영향 범위:

```text
01_사방넷원본수집
02_주문상세원장
원본 업로드 로직
상태 변경 감지 로직
오성 출고 전 취소 제동 로직
```

### 3. 오성 출고 전 취소 제동 해제 시나리오 추가

기존 규칙:

```text
오성 발주 파일생성완료 또는 업로드완료 후 주문상태가 취소/반품/교환 계열로 바뀌면 오류상태=확인필요, 오류유형=출고전취소, 제동여부=Y로 후속 처리를 막는다.
```

추가 보완:

```text
제동 이후 사람이 오성 창고에 출고 중단 가능 여부를 확인한 뒤 처리 결과를 기록한다.
```

추가 컬럼:

```text
제동처리결과
제동처리일시
제동처리자
제동처리메모
```

처리 결과:

```text
출고차단성공
출고차단실패
```

출고차단성공 시:

```text
사용여부 = 제외
제동여부 = N
오류상태 = 처리완료
송장수신상태 = 제외
사방넷송장업로드상태 = 제외
거래처회신상태 = 제외
ERP업로드상태 = 제외
마감대조상태 = 제외
```

출고차단실패 시:

```text
제동여부 = N
송장/사방넷/거래처/ERP 후속 흐름을 계속 진행
단, 처리메모에 출고차단실패 기록
```

영향 범위:

```text
02_주문상세원장
91_오류로그
후속 파일 생성 대상 필터
제동 해제 버튼
```

### 4. NS-3000 분리 + 치마형 키트 합포장 복합 규칙 명확화

문제 상황:

```text
NS-3000 수량 2~3개 분리 규칙과 치마형 키트 합포장 규칙이 동시에 적용될 때, 부속품을 어느 분리 행에 붙일지 모호했다.
```

확정 규칙:

```text
NS-3000 수량 2~3개 분리 시, 치마형 키트 등 합포장 대상 부속품은 분리된 첫 번째 NS-3000 행에만 결합한다.
나머지 NS-3000 분리 행에는 부속품을 복사하지 않는다.
```

예시:

```text
주문:
NS-3000 3개
치마형 키트 1개

출력:
1행: NS-3000 + 치마형 키트 합포장 코드
2행: NS-3000 단독
3행: NS-3000 단독
```

절대 금지:

```text
분리된 모든 NS-3000 행에 치마형 키트를 복사하는 것
```

영향 범위:

```text
오성 업로드 파일 생성
92_품목마스터
NS-3000 분리 로직
합포장 로직
```

### 5. 장기 보관 구조 메모 추가

확정 방향:

```text
운영 시트 = 풍림전자_매출관리원장
보관 시트 = 풍림전자_매출관리원장_보관_2026
```

운영 원칙:

```text
운영 원장은 현재 처리용이다.
완료된 오래된 데이터는 별도 보관 시트로 옮긴다.
보관용 시트는 일상 프로그램과 직접 연동하지 않는다.
```

향후 추가 개발:

```text
보관 처리 버튼
반기별/연도별 아카이브 정책
```
